# AGENTS.md — repository0 (the intentïon template)

AAIF-standard guidance for AI agents. Repo conventions are in `CLAUDE.md`;
estate-wide conventions in `../CLAUDE.md` (the `polycode-public` parent).

## What this is

**The template a fleet repo is cut from.** It carries the product (a Node/ESM
library skeleton) and *zero* engine machinery in its target end-state. A fleet
repo is created by cutting this template and giving it an `INTENT.md` (the fixed
point — what should exist); the **agentic-lib** engine then delivers that intent
via PRs.

## Licensing — IMPORTANT

**This repo stays MIT.** It is a template, so anything cut from it must be
unencumbered. Do **not** relicense it to AGPL. (Our IP — `agentic-lib`, the site,
marginalia — is AGPL; the template is deliberately permissive.)

## Build / test

- `npm test` / `npx vitest` — unit + system tests.
- `npx playwright test` — behaviour tests.
- Node 24+, ESM (`"type": "module"`).

## Target end-state (post-normalisation, PLAN_CODING_AGENT §7.9)

- `INTENT.md` (renamed from `MISSION.md`) — the fixed point.
- `AGENTS.md` (this file), `agentic-lib.toml` (slim: `[engine]`/`[caps]`/`[paths]`).
- **3 thin consumer workflows** pinning the engine:
  `on-intent.yml` / `on-review.yml` / `on-schedule.yml`, each
  `uses: polycode-public/agentic-lib/.github/workflows/transform.yml@v8`.
- Product only: `src/lib/main.js`, `tests/`, `vitest.config.js`,
  `playwright.config.js`, `eslint.config.js`. **No `agentic-lib` runtime dep.**
- Deleted: `MISSION_COMPLETE.md`, `library/` + `SOURCES.md`, `features/`, and the
  vendored `.github/agentic-lib/` + `.github/agents/` machinery (regenerated by the
  engine, not committed here).
